# KB-05 · 角色场景锁定语法

> **加载阶段**：阶段二（视觉开发）+ 阶段三（分镜提示词）。
> **用途**：定义如何在即梦提示词中使用 `@图片X` 参考图锚定来保证跨镜头一致性。

---

## 1. @图片锚定的本质

### 1-1 为什么要锚定

Seedance 2.0 模型在每次生成时记忆是孤立的，跨镜头时如果**没有显式锚定**，会出现：
- 同一角色换脸（面部五官微调）
- 同一角色换服装（颜色/款式漂移）
- 同一场景换布局（墙面/地面/光线重组）
- 动作衔接错位（上一个镜头停在左侧，下一个镜头出现在右侧）

### 1-2 锚定能解决什么

| 问题类型 | 锚定方案 |
|---|---|
| 角色面部一致性 | 挂 @图片 锁定面部参考 |
| 角色服装一致性 | 挂 @图片 锁定全身参考 |
| 场景构图一致性 | 挂场景 @图片 作为场景参考 |
| 跨镜头动作衔接 | `承接首帧@图片X` 锁定起始画面 |
| 镜头结束画面控制 | `尾帧@图片X` 锁定结束画面 |
| 光线色温一致性 | 挂灯光氛围 @图片 参考 |

---

## 2. 五种核心锁定语法

### 2-1 全锁定（适用于固定人物，不做任何变化）

```
X的五官、脸型、发型、服装全部锁定@图片1，全程不改变
```

**适用**：同一角色在多个生成单元中外形完全不变。

---

### 2-2 差异化锁定（保留部分，开放部分）

```
女孩五官、脸型锁定@图片1，但发型改为散发披肩，服装换为@图片5
```

**适用**：同一角色在不同场景有外形微调（如战斗状态 / 受伤状态）。

**语法规则**：
- 先写「锁定的部分 + @图片X」
- 后写「改变的部分 + 具体描述 / @图片Y」
- 永远不要省略"保留的部分"，否则 AI 会随机生成

---

### 2-3 样貌同 + 差异化（简化写法）

```
样貌同@图片1，但散发披肩，不戴眼镜，右侧嘴角有血渍，左手是机械臂
```

**适用**：基础外形一致，只有局部差异时的简化版写法。

---

### 2-4 否定式锁定（堵住 AI 的错误倾向）

```
她是亚洲人，不要生成欧美人形象
皮肤稍微粗糙有战斗痕迹，不要太过光滑
不要忘记左手的机械臂
不要忘记她左手戴着一个发旧的蓝色手套
```

**适用**：AI 有频繁的错误倾向需要显式堵截。

**写法原则**：
- 先正后反：先描述**正确应该有什么**，再补**不要出现什么**
- 越具体越好：`不要欧美人形象` > `不要错误生成`
- 针对性强：只堵 AI 真的会犯的错误，不要列超长否定清单

---

### 2-5 组合式锁定（最严密）

```
承接首帧@图片3，女孩五官锁定@图片1，服装锁定@图片5，
右手握蝴蝶刀（不要忘记），左手空着，
不要生成欧美人形象，不要忘记右脸颊的蝴蝶纹身
```

**适用**：关键镜头需要最严密的一致性控制（如面部大特写、关键打击瞬间）。

---

## 3. 场景锁定语法

### 3-1 四视图全面锁定（最完整）

```
场地四视图@图片3
```

**适用**：第一次进入某场景时，锁定场景的完整空间。

---

### 3-2 氛围参考锁定

```
公寓灯光氛围参考@图片1
场地风格参考，灯光氛围参考@图片5
```

**适用**：同一场景在不同时间/情绪下的光线微调。

---

### 3-3 位置关系锁定

```
两人的位置关系@图片4
承接首帧@图片4，保持两人站位关系
```

**适用**：多人镜头中角色站位的一致性。

---

### 3-4 场景与角色联合锁定

```
场景锁定@图片10（苔藓林全景），
香蕉猫样貌锁定@图片1，
刀盾狗样貌锁定@图片4，
两人站位：香蕉猫居画面左侧，刀盾狗跟在其右后方约 1 米
```

**适用**：新场景第一个生成单元的最严密锁定写法。

---

## 4. 首尾帧锚定语法（跨生成单元连接核心）

### 4-1 首帧锚定（锁定起点）

```
承接首帧@图片X，镜头（仰拍/俯拍/推近...），（动作描述）...
```

**作用**：
- 让本单元的起始画面严格匹配参考图
- 跨单元动作衔接的核心手段
- 复杂起手姿势不用文字描述，直接给图

**示例**：
```
承接首帧@图片3，镜头仰拍大特写，
女孩直视反派，冷风吹动发丝，她缓缓开口说话...
```

---

### 4-2 尾帧锚定（锁定终点）

```
（完整动作描述）...，尾帧@图片Y
```

**作用**：
- 让本单元的结束画面精确停在参考图上
- 为下一单元提供明确的承接起点
- 控制关键动作的完成状态

**示例**：
```
（打斗动作描述）...铁锤将蝴蝶推飞几米外，尾帧@图片2
```

---

### 4-3 首尾帧同时锚定（最稳）

```
承接首帧@图片X，（动作描述）...，尾帧@图片Y
```

**作用**：两端都锁定，中间动作最可控，是复杂动作段的标配。

**适用场景**：
- 跨单元的关键动作衔接
- 精确控制起止画面的格斗镜头
- 情绪转换的临界点

---

### 4-4 延续 vs 承接（两种衔接强度）

| 连接词 | 含义 | 使用场景 |
|---|---|---|
| 承接首帧@图片X | **严格**匹配，画面精确一致 | 跨片段动作衔接，要求无缝 |
| 延续首帧@图片X | **宽松**延续，场景状态延续即可 | 场景延续，动作可有差异 |
| 延续@图片X | **最宽松**，同场景再次出现 | 同场景多镜头间的一般引用 |

---

## 5. 反打/对切语法

### 5-1 反打首帧锚定

```
对切首帧@图片X 的反打镜头，过对方肩拍回来（注意不要越轴）
```

**适用**：对话场景的正反打切换。

### 5-2 注意轴线关系

所有反打镜头必须写明轴线关系（详见 KB-08 剪辑逻辑库）：

```
承接首帧@图片4 的反打镜头（注意不要越轴），
过铁锤关系仰拍蝴蝶脸部大特写
```

---

## 6. 叠化转场锚定

```
由首帧@图片X 这个画面无缝叠化转场至（新场景描述）
```

**适用**：
- 时间跳跃
- 回忆插入
- 场景转换
- 梦境/现实切换

**示例**：
```
由首帧@图片2 这个画面无缝叠化转场至地上一滩雨水的特写，
透过雨水反射，女孩的身影走过来并停下
```

---

## 7. 锚定的常见错误

### 错误一：用不存在的 @图片编号

```
❌ 承接首帧@图片7，...（但@图片7还没生成）
```

**正确做法**：先生成图片、登记到 @图片库，再写提示词。阶段三可以先用 `【待补图】` 占位，批量生成后回填。

### 错误二：跨场景混用编号

```
❌ 场景A 的镜头用 @图片1 锚定香蕉猫
❌ 场景B 的镜头改成 @图片3 锚定香蕉猫（同一角色不同编号）
```

**正确做法**：每个角色的 @图片编号全项目固定不变。

### 错误三：首帧锚定后不写动作

```
❌ 承接首帧@图片3。女孩。
```

**正确做法**：首帧锚定只是锁定起点，后续必须完整描述运镜、动作、场景等所有要素。首帧不能替代站位描述、虚实关系、焦点写法。

### 错误四：锚定了却写反描述

```
❌ 承接首帧@图片1（图中角色穿黑色），但她穿白色连衣裙...
```

**正确做法**：锚定图的视觉和描述必须一致。需要改变服装时用差异化锁定语法（见 §2-2）。

## 8. 锚定语法速查表

| 需求 | 语法 |
|---|---|
| 锁死角色全部外形 | `X锁定@图片N，全程不改变` |
| 部分锁死，部分改变 | `X五官锁定@图片N，发型改变，服装换为@图片M` |
| 简化版锁定 | `样貌同@图片N，但[差异描述]` |
| 堵住错误倾向 | `不要生成X，不要忘记Y` |
| 锁定场景空间 | `场地四视图@图片N` |
| 锁定光线氛围 | `灯光氛围参考@图片N` |
| 锁定起始画面 | `承接首帧@图片N` |
| 锁定结束画面 | `...尾帧@图片N` |
| 两端都锁定 | `承接首帧@图片N，...尾帧@图片M` |
| 宽松延续 | `延续首帧@图片N` / `延续@图片N` |
| 反打衔接 | `对切首帧@图片N的反打镜头（注意不要越轴）` |
| 叠化转场 | `由首帧@图片N这个画面无缝叠化转场至...` |

## 9. 锚定与其他要素的关系

### 9-1 锚定 ≠ 替代站位描述

即使挂了 @图片，仍然需要用文字说明：
- 镜头运动（运镜类型）
- 虚实关系（前景/中景/后景）
- 焦点位置（实焦主体）
- 动作描述（具体做什么）

**原因**：@图片只锁定外形/构图，不锁定动作、运镜、景深。

### 9-2 锚定 + 文字描述必须一致

如果挂了一张站立的参考图，就不要在文字中写"躺着"，否则模型会混乱。

### 9-3 锚定优先级

同一生成单元内多个锚定的优先级：
1. 首帧 > 全局锁定
2. 全局锁定 > 差异化锁定
3. 差异化锁定 > 否定式补充

即：如果首帧 @图片3 显示的是蓝色衣服，全局写的是 @图片1 红色衣服，模型会以首帧为准。
