# 即梦 Seedance 2.0 提示词终极手册（V6）

**视频时长定义**：
- 短视频 = 3分钟以内（180秒），本手册绝大多数规则基于此
- 长视频 = 3分钟以上，部分节奏规则有所放宽

---

## 零、底层写作规范（必须先读）

> 这三条规范是 Seedance 2.0 官方底层逻辑的直接体现，所有提示词必须遵守，优先级高于其他一切技巧。

### 规范一：纯中文写作，禁止中英混杂

Seedance 2.0 对中文原生语义的理解精度远高于英文或中英混合写法。英文单词夹在中文句子里，模型会切换到翻译模式解析，语义准确率大幅下降。

```
❌ 错误：dolly-in推镜，穿黑色oversize卫衣的dancer腾空起跳
✅ 正确：镜头缓慢向前推进，穿黑色宽松卫衣的街舞舞者屈膝蓄力腾空跳起
```

**唯一例外**：风格词（如 4K、60fps、16:9）因为是技术参数，可以保留英文数字格式。

---

### 规范二：权重优先级不可逆——运镜必须放最开头

Seedance 2.0 官方确认的权重优先级排序：

```
运镜/视角 > 动作时序 > 主体定义 > 场景环境 > 光影氛围 > 风格调性 > 细节约束
```

**写作顺序**：运镜（最高权重）→ 主体（谁 + 站在哪）→ 动作（做什么）→ 后续细节

> ⚠️ **常见困惑：动作时序权重排第二，为什么写作顺序却排在主体之后（第四位）？**
>
> 因为动作一定是某个主体发出的。模型在解析时，必须先知道"谁"，才能把动作正确归属到这个人身上。
> 如果先写动作再写主体，模型找不到动作发出者，会自由分配给随机对象，导致出片混乱。
>
> **权重排序 = 模型赋予信息的重要性**；**写作顺序 = 保证模型能读懂信息的逻辑链**。两者是不同维度，不冲突。

运镜放最开头，模型首先锁定镜头逻辑，出片率提升30%以上。

---

### 规范三：单个生成单元提示词长度控制在 120–280 字

> **此处"字"指中文字符数**，按正常输入法计字，不按词组数计算。

| 长度 | 问题 | 后果 |
|---|---|---|
| 少于120字 | 信息不足，随机性过高 | 模型自由发挥，结果不可控 |
| 120–280字 | 最优范围 | 核心权重集中，模型精准执行 |
| 超过280字 | 信息过多，权重稀释 | 核心指令被边缘信息淹没，效果降级 |

写完后数一遍字数，超出就删掉信息密度最低的描述，不足就补充动作细节或微表情。

## 一、【新增核心】站位描述体系

> ** 参考图只锁定角色外形，不锁定空间位置。没有站位描述，即梦会自由猜测人物在画面中的位置，多人镜头翻车率极高。**不管有没有参考图，每个多人镜头都必须写站位。**

### 站位描述的四个维度

**① 横轴位置（左中右）**

用画面三等分来定位每个人物：

```
女孩站在画面左侧1/3，boss站在画面右侧
两人各占画面一侧，中间留有空隙
```

**② 纵深层次（前景 / 中景 / 后景）**

明确谁在前、谁在后，以及虚实关系：

```
前景是虚化的反派背影，后景实焦是女孩
前景左侧是虚化的门框，中景是面向镜头的男主
```

**③ 相对距离**

```
两人相距约一臂距离，正面对峙
boss站在女孩正前方两步处，微微俯视
```

**④ 主次关系（谁是实焦主体）**

多人镜头中明确焦点在谁身上：

```
镜头焦点在女孩身上，boss身影虚化在后景
```

### 站位描述的写法位置

站位描述紧跟在主体外形描述之后，动作描述之前，独立成一句：

```
【运镜】固定中景，手持轻微晃动。
【主体】30岁亚洲男性，寸头，黑色练功服，白腰带。
        25岁亚洲女性，高马尾，红色短打功夫上衣，黑色武术裤。
【站位】两人相距两米正面对峙，男性居画面左侧，女性居右侧，镜头焦点对准两人中间。
【动作】男性右腿猛地抬起踢出旋踢，女性急速左闪……
```

### 过肩镜头（过肩反打）的站位写法

过肩镜头是最容易模糊的构图，必须明确：

```
镜头过男性右肩拍女性，男性右肩虚化占画面左下角1/5，
后景实焦是女性面部大特写，居画面右侧偏中，两人相距约一米。
```

### 单人镜头的站位写法

单人镜头也需要写站位，否则模型不知道主体在画面哪个位置：

```
女孩居画面中央偏左，背景是虚化的走廊，镜头焦点在她的面部。
```

## 二、【新增核心】虚实关系与景深三层次

> Seedance 理解"虚实"的前提是你明确写出来。没有写虚化的元素，模型默认全部实焦，景深感消失。

### 景深三层次描述公式

```
前景（虚化/实焦）+ 中景（主体，通常实焦）+ 后景（虚化/实焦）
```

**完整示例：**

```
前景是虚化的反派蜷缩的身体，
中景是实焦的女孩侧面近景，镜头焦点在她的面部，
后景是虚化的废旧铁轨和杂草。
```

### 焦点转移（变焦）的写法

```
镜头焦点从前景虚化的机械臂逐渐后拉变实，
同时后景的女孩面部从虚化逐渐变为实焦大特写。
```

### 虚实的常见用途

| 用途 | 写法 |
|---|---|
| 强调主体 | 前后景虚化，主体实焦 |
| 制造层次感 | 前景实焦道具/手，后景虚化人物 |
| 转移注意力 | 焦点从A转移到B（从虚变实） |
| 过肩/过人关系 | 前景过肩人物虚化，后景对话人物实焦 |

## 三、【新增核心】生成时长 vs 成片时长

> ** 手册里凡是标注了性质的时长数字，会明确写"（成片时长）"或"（生成时长）"。没有特别标注时，以本章规则判断。

### 两个时长的本质区别

| 概念 | 定义 | 是谁决定的 |
|---|---|---|
| **成片时长** | 最终视频里这段剪辑留多久 | 你在后期剪辑决定 |
| **生成时长** | 提交给即梦的生成参数填几秒 | 你在提交时填写，最低4秒、最高15秒 |

### 两条规则，各管不同场景，不冲突

**规则A（成片3秒以上的镜头）：生成时长 = 成片时长 + 2秒头尾裁剪冗余**

即梦生成的是"毛素材"，头尾两帧往往不稳定（动作"还没准备好"或"突然停住"）。生成比成片多2秒，后期截取中间稳定段。

```
成片需要一个5秒的特写（成片时长）
↓
生成时长填7秒（5+2，加头尾裁剪冗余）
↓
后期截取最稳定的5秒使用
```

**规则B（成片2秒以下的镜头）：永远打包，不单独提交**

成片≤2秒的极短镜头，即使单独提交也只能生成4秒（平台最低生成时长），浪费生成配额。正确做法是并入相邻镜头一起提交。

```
成片需要一个1.5秒的闪切（成片时长）
↓
不单独提交，并入相邻镜头
↓
一次提交，后期从整段素材里截取该1.5秒
```

### 生成时长参考表

> 下表左列均为**成片时长**，右列均为**生成时长（填写参数）**。

| 成片时长 | 处理方式 | 生成时长填写 |
|---|---|---|
| ≤ 2秒 | **并入相邻镜头，不单独提交** | — |
| 3–4秒 | 单独提交，加2秒冗余 | 5–6秒 |
| 5–7秒 | 单独提交，加2秒冗余 | 7–9秒 |
| 8–11秒 | 单独提交，加2秒冗余 | 10–13秒 |
| ≥ 12秒 | 单独提交，填推荐上限 | **13秒**（注） |

> **注（边界情况）**：成片时长超过11秒时，按规则A应填14秒以上，但生成参数推荐上限为13秒，规则A在此区间自动失效。此时接受约1秒冗余，直接填13秒即可。不要机械套用"加2秒"公式。

**铁律**：生成参数不填15秒（官方最高上限），推荐填写上限为13秒。这2秒是相对于平台上限保留的**平台安全余量**，与规则A的头尾裁剪冗余是两个不同概念，不要混淆。

## 四、【新增核心】生成单元的三种合并/拆分策略

> ⚠️ **V6 重要修正**：本章将 V5 的"打包提交"拆分为两个独立概念——「无缝合并」和「分镜打包」，并将拆分策略从两种情况修正为三种情况。

### 核心概念区分

| 概念 | 含义 | 使用格式 | 目的 |
|---|---|---|---|
| **无缝合并** | 景别和运镜完全不变，多个动作写在同一段描述里，无剪辑点 | 直接用"随后""紧接着"串联 | 让AI生成连贯动作 |
| **分镜打包** | 景别或运镜有变化，一次提交包含多个镜头，让即梦自动生成剪辑节点 | 用"镜头一（Xs）""镜头二（Xs）"格式 | 减少提交次数，节省配额 |
| **独立提交** | 动作激烈或复杂，算力不够分，必须拆开分别提交 | 各自单独提交 | 保证动作质量 |

---

### 三种情况，对号入座

**情况一：景别 + 运镜都不变 → 无缝合并**

同一个镜头位置，同一个运镜风格，只是动作有先后顺序：

```
固定中景，手持轻微晃动。
（主体/站位/场景描述...）
女孩蹲下来检查伤口，随后抬起头扫视四周，紧接着缓缓起身，手按刀柄保持警戒。
→ 一次提交，生成时长按总成片时长+2秒填写
```

**情况二：景别或运镜有变化，但动作不激烈 → 分镜打包**

用"镜头一/二"格式，一次提交，让即梦在切换点自动生成剪辑：

```
总计两个镜头。
镜头一（6秒，成片时长）：固定中景，手持轻微晃动。
  女孩在审讯室里缓缓走近，绕到反派椅子后方，双手搭在椅背上低头俯视。
镜头二（5秒，成片时长）：承接上个镜头，仰拍大特写，
  切近女孩面部，她冷冷开口，嘴唇轻动说话。
→ 成片总时长11秒，生成时长填13秒（11+2，受平台上限取整）
→ 一次提交
```

> **分镜打包的时长计算**：各镜头成片时长之和 ≤ 11秒，加2秒冗余后≤ 13秒（生成时长），符合参数上限。

**情况三：景别或运镜变化，且含激烈动作 → 必须独立提交**

激烈打斗、高速追逐等复杂动作需要大量算力，和其他镜头打包会导致两边质量都下降：

```
激烈打斗核心段 → 单独提交，生成时长按规则A填写
定场/收尾特写  → 单独提交，生成时长按规则A填写
→ 后期剪辑拼接
```

---

### 极短镜头（成片≤2秒）的处理

极短镜头永远不单独提交，并入最近的相邻镜头：

```
❌ 单独提交一个成片1.5秒的闪切镜头 → 浪费生成配额
✅ 把该1.5秒闪切并入相邻镜头的分镜打包或无缝合并里 → 一次提交，后期截取
```

## 五、两种运镜的本质区别

> 搞清楚这个，运镜翻车问题迎刃而解。

### 路径型运镜 ← AI 当作轨迹指令执行

描述镜头的具体移动路径和终点，AI 需要精确计算镜头轨迹。

```
例：镜头从左侧以45度弧线推进，同时向上摇，最终停在人物右肩特写位置
例：镜头从远景缓慢推至中全景，人物停稳后固定
```

**特点**：占用大量算力；动作越复杂，路径型运镜越容易导致动作变形。

### 质感型运镜 ← AI 当作风格标签理解

描述镜头的整体氛围和质感，不规定具体轨迹，AI 自主解释执行。

```
例：手持摄影，全程晃动，剧烈呼吸感
例：镜头全程环绕推拉，纪录片写实风格
```

**特点**：几乎不占算力；算力全部释放给动作本身。

### 核心使用规则

| 动作复杂度 | 运镜策略 |
|---|---|
| 极复杂（激烈打斗/高难度技巧） | 固定镜头 + 质感型运镜 |
| 中等（走动/追击/对峙） | 路径型 + 质感型叠加 |
| 极简（静止/对话/情绪） | **固定为主，可叠加质感型**（如"固定中景，手持轻微晃动"） |

> **说明**：对话场景虽属"极简"，但统一推荐使用固定镜头（而非路径型推拉），原因是路径型运镜会分散模型对嘴型同步和面部表情的处理算力。"固定+质感型晃动"是对话场景的黄金组合。

**黄金组合公式**：

```
路径型运镜 → 只放在 clip 的起点（静止状态）或终点（动作完成后）
质感型运镜 → 贯穿全程，作为整体风格标签叠加
```

## 六、【新增核心】轴线关系（180度法则）

> 这是影视拍摄中最基础的空间逻辑，但即梦不会自动遵守，必须在提示词里明确控制。

### 什么是轴线

两个角色之间存在一条虚拟的"轴线"（对话轴/运动轴）。在同一场景的所有镜头里，摄影机必须保持在轴线同一侧拍摄，否则两人的左右关系会发生翻转，观众感知到空间错乱。

### 越轴的典型错误

```
镜头A：男性在画面左侧，女性在画面右侧，男性看向右方（看着女性）
镜头B（越轴）：男性在画面右侧，女性在画面左侧，男性却还在看向右方
→ 观众感知：两人突然"穿越"到对方位置，空间错乱
```

### 如何在提示词里控制轴线

**方法一：每个镜头明确写各人物在画面左右的位置**

```
镜头一：男性居画面左侧，女性居画面右侧，男性看向右（看向女性方向）
镜头二（反打）：女性居画面左侧，男性居画面右侧，
注意：此处是反打，男性在右，女性在左，轴线关系不变
```

**方法二：使用「过肩反打」时明确说明**

```
承接上个镜头，过男性右肩反打女性（注意不要越轴），
男性右肩虚化在画面左下角，女性实焦居画面右侧中央
```

**方法三：在提示词中直接写"注意不要越轴"**

在多人对话或打斗的连续镜头中，可以直接在结尾加一句：

```
注意保持轴线关系，不要越轴
```

## 七、【新增核心】首尾帧参考图锚定

> 这是成片里最强的一致性手段。

### 什么是首尾帧锚定

用参考图明确指定一个生成单元的**起始画面**和**结束画面**，模型会以这两个画面为锚点，生成中间的过渡动作。

```
承接首帧@图片1，铁锤将蝴蝶推飞几米外……（中间动作）……尾帧@图片2。
```

### 首帧锚定 vs 尾帧锚定

| 用途 | 写法 |
|---|---|
| 锁定起始状态 | 「承接首帧@图片X，…（动作描述）」 |
| 锁定结束状态 | 「…（动作描述）…到尾帧@图片X」 |
| 两端都锁定 | 「承接首帧@图片X，…（动作描述）…，尾帧@图片Y」 |

### 首帧锚定解决的核心问题

1. **跨生成单元的动作衔接**：上一个单元生成到某个画面，下一个单元的起点就是同一张参考图，动作连贯性大幅提升
2. **保证打斗姿势的合理性**：不用文字描述复杂的起手姿势，直接给一张参考图让模型对齐
3. **场景构图一致性**：同一个场景反复出现时，每次都锚定同一张全景参考图作为首帧

### 首尾帧描述的注意事项

- 首帧参考图必须和上一个生成单元的结束画面匹配，不能跳步
- 参考图编号在整份提示词内要统一管理，不能混用
- 首帧锚定不能替代站位描述——仍然需要用文字说明镜头运动、虚实关系和焦点

## 八、【新增核心】升格镜头（慢动作）

> 升格是强化冲击瞬间的标准手段，但需要正确使用，否则全片都是慢动作会拖垮节奏。

### 升格的写法

直接在运镜/视角部分写明：

```
升格镜头，固定特写，手持轻微晃动
```

或者在动作描述中写明：

```
女孩起身，镜头切入升格，她将刀干练收起，头发随动作轻轻飘起，每一根发丝都清晰可见
```

### 升格的适用场景

| 场景 | 是否适合升格 |
|---|---|
| 关键打击瞬间（拳击命中/刀刺入） | ✅ 强烈推荐 |
| 情绪特写（眨眼/微笑/泪水） | ✅ 推荐 |
| 走廊行走、过渡镜头 | ❌ 不推荐，拖慢节奏 |
| 激烈连续打斗段 | ❌ 不推荐，会让打斗失去紧迫感 |
| 收尾定格（主角转身离开） | ✅ 适合 |

### 升格不能替代场景描述

升格只是拍摄速率，其他所有描述（站位、构图、虚实、动作）仍然必须写完整。

## 九、【新增核心】转场描述

> 片段内部的转场（不是后期剪辑的转场）可以在提示词里明确描述，即梦会尝试执行。

### 三种常用转场的写法

**硬切（直接切换，最常用）**

不需要特别描述，即梦默认在镜头切换处硬切。分镜打包时直接写：

```
镜头一（6秒，成片时长）：…
镜头二（5秒，成片时长）：承接上个镜头，…
```

**叠化转场（Dissolve）**

```
由首帧@图片1这个画面无缝叠化转场至地面一滩雨水的特写
```

适用于时间跳跃、回忆闪回、场景转换。

**甩镜转场**

```
镜头快速向右甩出，画面模糊的瞬间切入新场景
```

### 转场描述的位置

写在前一个镜头结束、后一个镜头开始的衔接处：

```
镜头一（5秒，成片时长）：…女孩转身出画。
→ 叠化转场
镜头二（6秒，成片时长）：叠化进入夜晚场景，女孩站在建筑门口……
```

## 十、【新增核心】音效与配乐指令

> 即梦支持在提示词中指定音频方案。明确写出来，比让模型自动生成的效果更可控。

### 四种音频方案

| 方案 | 写法 | 适用场景 |
|---|---|---|
| 只有音效，无配乐 | 「只需要音效，无配乐」 | 打斗、紧张对话、写实纪录风格 |
| 只有配乐，无音效 | 「只需要背景配乐，无音效」 | 文艺片、情绪段落 |
| 音效 + 配乐 | 「音效与配乐同时存在，配乐低混，音效突出」 | 动作场面的戏剧性增强 |
| 完全静音 | 「无音效无配乐，完全静音」 | 用于后期自行添加音轨 |

### 音效描述的写法

可以描述具体的音效内容，而不只是说"有音效"：

```
脚步声由远及近，刀刃划过金属的摩擦声，玻璃破碎声
```

```
雨水溅落的声音，沉闷的击打声，反派倒地的声音
```

### 有音频参考时的对话写法

挂上音频参考（`@音频X`）后可以直接写台词：

```
女孩冷冷说：我再问你最后一遍，谁指使你们来的。
```

**没有 `@音频` 时，不写台词文字，只写嘴部动作：**

```
女孩开口说话，嘴唇随说话节奏轻轻动着，眉头微皱，语气激烈
```

## 十一、完整写作公式（含站位和构图）

```
【第一位】运镜/视角（最高权重，必须放最开头）
  → 例：固定中景，手持强烈晃动，升格

【第二位】主体外形描述（完整，从头到脚，多角色按主次排序）
  → 例：30岁中国男性，寸头，浓眉方颌，穿黑色棉麻练功服，束白腰带，黑色千层底布鞋
  ※ 动作时序权重排第二，但主体必须在动作之前出现，否则模型找不到动作发出者

【第三位】站位与构图（横轴位置 + 纵深层次 + 虚实关系 + 相对距离）
  → 例：男性居画面左侧，女性居画面右侧，两人相距两米正面对峙，
         前景左角是虚化的废弃铁架，镜头焦点对准两人

【第四位】大动作（时间顺序分步写，从起点到终点，不只给静态结果）
  → 例：男性右腿猛地抬起踢出旋踢，女性急速左闪，旋踢擦过耳侧

【第五位】微表情 / 微动作细节（消弭AI僵硬感）
  → 例：胸口急促起伏，鼻翼微张，发丝被气流吹乱，自然眨眼

【第六位】对话状态处理
  → 有音频：直接写台词；无音频：只写嘴部动作状态

【第七位】场景 + 环境描述（优先使用短场景锚定）
  → 例：深山青石武台，四周苍松环绕，薄雾低伏，清晨冷蓝天光，逆光轮廓清晰

【第八位】光影氛围
  → 例：逆光轮廓清晰，冷蓝色天光，高对比

【第九位】风格调性
  → 例：写实动作风格，纪录片质感，高对比冷色调

【第十位】质量约束（人物必加）
  → 面部五官清晰稳定不变形，眼睛嘴型不扭曲，人体结构正常，比例自然，动作连贯不僵硬

【最后】音效指令 + 生成时长 + 比例
  → 只需要音效，无配乐，9秒（生成时长），16:9
  ※ 此处填写的是生成时长（成片时长+2秒头尾裁剪冗余），不是成片时长
```

## 十二、完整工作流（七步骤）

### 步骤一：建立固定要素库

所有不变信息提前标准化，这是跨生成单元一致性的根基。

```
【固定要素库 - 人物A：蝴蝶】
25岁亚洲女性，右脸颊有蝴蝶纹身，高马尾，利眼高颧骨，
穿黑色战术上衣，黑色紧身裤，右手惯用蝴蝶刀。

【固定要素库 - 场景：组织基地走廊】
夜晚，工业感走廊，左侧是带玻璃的实验室，
荧光灯冷白光打下来，地面有积水反光，
走廊尽头墙上是蝴蝶logo。
```

---

### 步骤二：按动作/情绪单元拆叙事分镜

一件完整的事 / 一个完整动作 = 一个叙事分镜。

| 视频节奏 | 100秒合理拆分 | 单镜成片时长 | 适用场景 |
|---|---|---|---|
| 慢节奏文艺片 | 3–6 个生成单元 | 10–13秒 | 情感短片 |
| 中节奏剧情片 | 7–10 个生成单元 | 8–12秒 | 故事短片 |
| 快节奏动作片 | 12–15 个生成单元 | 4–8秒 | 打斗、快剪预告 |

> **表中时长均为成片时长**，生成时长请按规则A加2秒（上限13秒）。

**拆分判断标准**：景别变了 / 运镜变了 / 场景转了 / 情绪转了 → 必须拆叙事分镜。

---

### 步骤三：补镜头语言

**景别选择**：开场定场 → 全景；人物全身 → 中全景；对话上半身 → 中景；细节情绪 → 近景/特写

**运镜选择**：复杂动作 → 固定+质感型；对话/情绪 → 固定为主；高潮特写 → 倾斜+仰拍+上摇

---

### 步骤四：合并判断——叙事分镜 → 生成单元

对每两个相邻叙事分镜，判断属于以下哪种情况：

```
情况一：景别 + 运镜都不变
→ 无缝合并：用"随后""紧接着"串联写入同一段描述，不用"镜头一/二"格式

情况二：景别或运镜有变化，且动作不激烈
→ 分镜打包：用"镜头一（Xs，成片时长）""镜头二（Xs，成片时长）"格式，
   一次提交，时长之和（成片时长）≤ 11秒，生成时长填（时长之和+2秒）≤ 13秒

情况三：景别或运镜有变化，且含激烈动作
→ 独立提交：各自单独提交，后期剪辑拼接
```

**分镜打包写法：**

```
总计两个镜头。
镜头一（6秒，成片时长）：固定中景，手持轻微晃动。…
镜头二（5秒，成片时长）：承接上个镜头，仰拍大特写。…
→ 成片总时长11秒，生成时长填13秒（=11+2，取平台上限）
→ 一次提交
```

---

### 步骤五：补全站位与构图

每个多人镜头，在主体描述后面加一句站位描述。单人镜头写清楚主体在画面哪个位置。

---

### 步骤六：检查轴线、首尾帧、虚实

- 同一场景的正打反打，左右关系是否越轴？
- 需要连接上一个生成单元的动作，是否挂了首帧参考图？
- 多人镜头是否写了哪里实焦、哪里虚化？

---

### 步骤七：四项最终检查

1. **一致性**：同一人物/场景，参考图与短锚定是否稳定？首次建立或跑偏修正时是否使用完整锁定描述？
2. **衔接**：下一个生成单元起点 = 上一个终点，不跳步。
3. **生成时长**：每个生成单元填写的是**生成时长**（成片时长+2秒，最高13秒），已留头尾裁剪冗余？
4. **权重顺序**：运镜在最开头，音效/生成时长/比例放最后？

---

## 十三、跨生成单元一致性：四个维度

### 维度一：人物一致性（选一种方案，全程不混用）

**方案 A：纯文字锁定**
无参考图或首次建立时，同一角色使用完整锁定描述；后续常规镜头切换为短锚定描述，避免提示词被资产卡淹没。

**方案 B：参考图锚定**
每一个涉及该人物的生成单元都挂同一张参考图（`@图片X`），并配合 40-70 字短锚定描述。

⚠️ 无参考图时绝对不能用"人物同上"——应使用完整锁定描述。

### 维度二：场景光线一致性

同一场景同一时间段，优先使用短场景锚定 + 稳定光线方向；首次建立、无参考图或场景跑偏时再使用完整场景锁定描述。

### 维度三：动作衔接

```
✅ 正确：
生成单元A（终点）：…走到木门口，停下，右手扶住木门把手
生成单元B（起点）：女生右手握住木门把手，轻轻向内推开，侧身走进店内
```

### 维度四：跨场景切换

新生成单元开头必须明确交代新的时间/地点，否则模型沿用上一个的光线和环境。

---

## 十四、动感制造：拆分 + 专属特写生成单元

### 专属特写生成单元是什么

在关键动作发生后，专门切入一个生成单元，只拍该动作最具冲击力的局部细节：

- 拳头命中瞬间的面部反应
- 脚踝横扫破开地面雨水的特写
- 人物稳住后眼神的大特写

这个生成单元本身不推进剧情，但它创造了景别跳跃，这个跳跃本身就是动感来源。

### 标准动感拆分模板

> 下表括号内均为**成片时长**，提交时每个单元生成时长 = 成片时长 + 2秒。

```
生成单元1（成片4秒，生成6秒）· 定场起手  → 路径型推镜（静止承接）
生成单元2（成片4秒，生成6秒）· 交手核心  → 固定 + 质感晃动（算力全给动作）
生成单元3（成片3秒，生成5秒）· 冲击特写  → 仰拍/倾斜 + 升格（景别跳跃）
```

---

## 十五、不同题材的分镜策略

### 慢节奏文艺片

- 3–6个生成单元，长镜头保持情绪连贯
- 缓慢推/拉/摇（路径型为主），少用质感晃动
- 提示词重点：微表情、微动作、光影氛围、升格特写

### 快节奏动作片

- 12–15个生成单元，一个小动作一个生成单元
- 激烈打斗段：固定+质感晃动；定场/收尾：路径型推拉
- 打斗高潮后必插专属特写生成单元

### 中节奏故事剧情片

- 7–10个生成单元，按叙事节点切割
- 对话用固定中景（可叠加手持轻微晃动），转场用摇镜，情绪升级用推镜
- 关键情绪转折点后插入一个近景特写生成单元

### 营销广告短片

- 第一个生成单元4–6秒（成片时长）直接放核心钩子
- 产品外形描述和品牌文字在所有生成单元完全一致

## 十六、短视频节奏铁律（3分钟以内全部适用）

### 铁律一：第一帧必须有人，第一秒必须有动作

短视频没有预热。观众在前3秒决定要不要继续看。把环境信息塞进场景描述字段，不单独开空镜生成单元。

### 铁律二：空镜时长上限 1–2 秒（成片时长），且必须并入相邻镜头

独立空镜成片时长≤2秒，按规则B必须并入相邻生成单元，不可单独提交。更好的方案是把空镜元素写进第一个有人生成单元的场景描述里。

### 铁律三：试探性动作直接删掉

```
❌ 生成单元（成片5秒）：男性探出试探性刺拳，女性侧头闪开，两人重新站定
✅ 生成单元（成片3秒）：男性率先爆发，右腿猛踢女性腹部，女性硬格挡，冲击力逼退一步
```

### 铁律四：收尾最多 3 秒（成片时长）

```
❌ 收尾（成片5秒）：女性缓缓起身，整理马尾，低头最后看了一眼，转身缓步离开
✅ 收尾（成片3秒）：女性起身，俯视倒地男性，转身出画，镜头缓拉固定
```

### 各类型生成单元时长速查表

> 下表均为**成片时长**，提交生成时长请在此基础上加2秒（不超过13秒）。

| 生成单元类型 | 推荐成片时长 | 成片绝对上限 |
|---|---|---|
| 独立空镜 | **不可单独提交，必须并入相邻单元** | — |
| 定场/对峙（有动态） | 3–4秒 | 5秒 |
| 单次交手动作 | 3–5秒 | 6秒 |
| 专属特写 | 3–4秒 | 5秒 |
| 重击连招 | 5–6秒 | 8秒 |
| 收尾定格 | 2–3秒 | 4秒 |

## 十八、终版完整检查清单（V6）

### 单生成单元层面

- ✅ 提示词是否纯中文？有无中英混杂？
- ✅ 提示词字数是否在 120–280 **字**（中文字符数）之间？
- ✅ **运镜/视角在最开头**，音效/生成时长/比例在最后？
- ✅ 主体外形描述完整（年龄/五官/服装/特征），放在运镜之后？
- ✅ **★ 站位与构图是否写清楚？（横轴位置 + 纵深层次 + 虚实关系）**
- ✅ 动作按时间顺序分步写，从起点到终点，不只给静态结果？
- ✅ 已加微表情/生理细节，消弭AI僵硬感？
- ✅ 运镜类型正确：激烈动作用固定+质感型；对话/情绪用固定为主；简单走动用路径型+质感型叠加？
- ✅ 路径型运镜只放在起点或终点，激烈动作过程中不出现路径型？
- ✅ **★ 最后填写的是生成时长（成片时长+2秒头尾裁剪冗余，上限13秒），不是成片时长？**
- ✅ **★ 成片≤2秒的极短镜头是否已并入相邻单元，而不是单独生成？**
- ✅ **★ 需要升格的瞬间是否已在运镜位置写明"升格镜头"？**
- ✅ **★ 需要叠化转场的地方是否已写明"叠化转场"？**

## 十九、核心心法总结（V6）

**写作规范**：纯中文；120–240字（字符数）；运镜最开头锁权重。

**公式顺序**：运镜 → 主体短锚定（首次建立/无图/跑偏修正才用完整锁定）→ **站位与构图** → 大动作 → 必要微细节 → 对话状态 → 场景短锚定 → 光影 → 风格 → 精简质量约束 → 音效指令 → **生成时长**（成片时长+2秒）→ 比例。

**三种合并策略**：景别+运镜都不变→无缝合并；景别/运镜变化且动作不激烈→分镜打包（"镜头一/二"格式，成片总时长≤11秒）；景别/运镜变化且含激烈动作→独立提交。

**站位描述**：横轴位置（左/中/右）+ 纵深层次（前景/中景/后景）+ 虚实关系 + 相对距离。不管有没有参考图，每个多人镜头都必须写。

**轴线关系**：同场景正打反打的左右关系保持一致，越轴导致空间错乱。多人对话/打斗连续镜头主动写"注意不要越轴"。

**虚实描述**：主动写哪里实焦、哪里虚化、焦点在谁身上。没写就默认全实焦，景深感消失。

**两种时长，必须区分**：
- **成片时长**：后期剪辑留多久，用于规划叙事节奏
- **生成时长**：提交给即梦的参数，= 成片时长 + 2秒头尾裁剪冗余（上限13秒）
- 成片超过11秒时，规则A边界失效，直接填13秒即可

**两种"冗余"，含义不同**：
- 头尾裁剪冗余：生成比成片多2秒，用于截取稳定段（规则A）
- 平台安全余量：生成时长推荐填13秒而非官方上限15秒，留2秒安全空间

**分镜打包时长计算**：各镜头成片时长之和 ≤ 11秒，生成时长填（之和+2秒）≤ 13秒。

**首尾帧锚定**：`承接首帧@图片X` 锁定起点，`尾帧@图片X` 锁定终点，两端都锁定最稳。

**升格**：写在运镜位置，只用在冲击瞬间/情绪特写，不用于走路过渡。

**对话运镜**：固定中景为主，可叠加手持轻微晃动，不用路径型推拉（路径型会抢占嘴型同步的算力）。

**转场**：叠化写"无缝叠化转场至……"；硬切不需要特别描述。

**音效**：每个生成单元结尾明确写音频方案（只要音效/只要配乐/完全静音）。

**一致性**：参考图全挂 + 短锚定稳定；完整锁定只在首次建立、无图或跑偏修正时使用，绝不写"同上"。

**动感**：不靠单元内部堆运镜，靠拆分+景别跳跃+专属特写+升格的节奏组合。

**节奏**：第一帧有人，第一秒有动作，空镜并入相邻单元不单独提交，试探性动作删掉，收尾成片≤3秒。
